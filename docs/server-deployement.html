SSL Server Setup

Setup the server using the followings steps:

[0] Setup SSH key on the server

We'll make the assumption will be using an OVH server (pretty much the same for other suppliers). First move to your local store for the ssh keys (for instance /Users/pc/qx/ssh/qxloop).

ssh-keygen -b 4096

-b 4096 means that we want 4096 bit long key

This command prompts for passphrase. It also prompts for a filename (choose something meaningful, like qxloop-front).

this command generate two files: xx and xx.pub (private and public key). Public key is to be copied on the server admin panel (User account > Services > SSH Key > Create a new key). Don't forget to include this key on any new server OS installation.

[1] Login on the server over SSH

For the following, we assume that:

  • The (local, i.e. on client/dev machine) location of the private SSH key is ~/qx/ssh/ovh-server.ssh , from previsou step.
  • the user name is root. (NOTE: on OVH Server with Ubuntu installed, ubuntu is preferred)
  • The the server address (DNS) is ns352508.ip-46-105-114.eu as advertised by the dedicated servers provider.
ssh -i ~/qx/ssh/ovh-server.ssh root@ns352508.ip-46-105-114.eu

This command is resolving identity directly from private key (allow to use multiple SSH-key on the same dev machine).

NOTE: you'll be prompted to enter pass-phrase (see previous step).

NOTE2: you'll be prompted to enter pass-phrase (see previous step).

[2] Install the JVM on the server

Once connected, switch to sudo mode, using:

sudo -i

Then, good thing is to always update your system first before you do anything else. Run the following commands (in sudo mode, remember you're now sudo):

apt-get update && apt-get upgrade

Then install the latest version of JAVA using:

apt-get install default-jdk
